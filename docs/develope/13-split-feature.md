# 任務拆分功能

## 1. 功能概述

將單一任務拆分成多個獨立任務，支援手動分割內容。

## 2. 使用情境

| 情境 | 說明 |
|------|------|
| 任務過於龐大 | 一個 Epic 任務需要拆分為多個 Story |
| 多人同時處理 | 需要將工作分配給不同人 |
| 複雜度提升 | 發現任務比預期複雜，需要細分 |

## 3. 拆分方式

### 3.1 手動分割內容

用戶在文字編輯器中使用分隔標記來定義分割點：

```
---
```

**範例：**
```
這是任務的第一部分內容。
這裡描述了 XXX 功能的需求。
---
這是任務的第二部分內容。
這裡描述了 YYY 功能的實作。
---
這是任務的第三部分內容。
這裡描述了 ZZZ 功能的測試。
```

**拆分結果：**
- 新任務 1: 第一部分內容
- 新任務 2: 第二部分內容
- 新任務 3: 第三部分內容

## 4. 商業規則

### 4.1 拆分條件
- 只有根任務可以拆分（子任務不可拆分）
- 拆分後至少產生 2 個任務
- 拆分後最多產生 20 個任務

### 4.2 內容分配

| 欄位 | 處理方式 |
|------|----------|
| title | 每個新任務標題 = 原標題 + 序號 |
| description | 按分隔符號分割 |
| priority | 可為每個新任務单独設定，預設繼承原任務 |
| status | 預設為 BACKLOG |
| assigneeId | 預設為 null（空），由用戶後續分配 |
| projectId | 與原任務相同 |
| createdById | 與原任務相同 |
| dueDate | 預設為 null，可後續設定 |

### 4.3 關聯處理

| 關聯 | 處理方式 |
|------|----------|
| 子任務 | 刪除（隨原任務刪除） |
| 依賴 | 保留（指向第一個新任務） |
| 留言 | 全部移到第一個新任務 |
| 負責人 | 不自動分配 |

### 4.4 拆分後
- 原任務刪除
- 建立 N 個新任務
- 第一個新任務保留原任務 ID（可選，視需求）

## 6. API 實作要點

### 6.1 分割邏輯
```typescript
function splitDescription(description: string, separator = "---") {
  return description
    .split(separator)
    .map(s => s.trim())
    .filter(s => s.length > 0);
}
```

### 6.2 拆分端點
```typescript
// POST /api/tasks/:id/split

// 1. 驗證任務存在且為根任務
// 2. 解析分割點
// 3. 在 transaction 中：
//    - 建立多個新任務
//    - 複製留言到第一個新任務
//    - 重建依賴到第一個新任務
//    - 刪除原任務
```

### 6.3 Transaction 範例
```typescript
await prisma.$transaction(async (tx) => {
  // 1. 取得原任務的留言和依賴
  const originalComments = await tx.taskComment.findMany({ 
    where: { taskId: originalTaskId } 
  });
  const originalDeps = await tx.taskDependency.findMany({ 
    where: { taskId: originalTaskId } 
  });
  
  // 2. 解析分割
  const descriptions = splitDescription(data.splits);
  
  // 3. 建立新任務
  const newTasks = [];
  for (let i = 0; i < data.splits.length; i++) {
    const newTask = await tx.task.create({
      data: {
        title: data.splits[i].title || `${originalTask.title} (${i + 1}/${data.splits.length})`,
        description: data.splits[i].description,
        priority: data.splits[i].priority || originalTask.priority,
        status: "BACKLOG",
        projectId: originalTask.projectId,
        createdById: originalTask.createdById,
        orderIndex: originalTask.orderIndex + i
      }
    });
    newTasks.push(newTask);
  }
  
  // 4. 複製留言到第一個新任務
  for (const comment of originalComments) {
    await tx.taskComment.create({
      data: { ...comment, taskId: newTasks[0].id }
    });
  }
  
  // 5. 重建依賴到第一個新任務
  for (const dep of originalDeps) {
    await tx.taskDependency.create({
      data: {
        taskId: newTasks[0].id,
        dependsOnId: dep.dependsOnId
      }
    });
  }
  
  // 6. 刪除原任務（子任務會級聯刪除）
  await tx.task.delete({ where: { id: originalTaskId } });
});
```

## 8. 測試案例

| 案例 | 預期行為 |
|------|----------|
| 使用 --- 分割為 3 個任務 | 正確產生 3 個新任務 |
| 分割子任務 | 返回錯誤，子任務不可拆分 |
| 只有一個分割段 | 返回錯誤，至少需要 2 個 |
| 分割後超過 20 個任務 | 返回錯誤 |
| 原任務有留言 | 留言移到第一個新任務 |
| 原任務有依賴 | 依賴移到第一個新任務 |
