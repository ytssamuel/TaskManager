# 任務合併功能

## 1. 功能概述

將多個任務合併為單一任務，保留所有有價值的資料，並解決可能存在的衝突。

## 2. 使用情境

| 情境 | 說明 |
|------|------|
| 多人建立重複任務 | 發現多個相似任務，合併為一個 |
| 任務過於細碎 | 多個小任務應該是同一個大任務 |
| 規劃變更 | 原本分開的任務現在證明是同一件事 |

## 3. 合併策略

### 3.1 保留的資料

| 資料類型 | 合併策略 |
|----------|----------|
| 標題 (title) | 用戶選擇（預設第一個任務的標題） |
| 描述 (description) | 串接，用 `---` 分隔各任務描述 |
| 優先級 (priority) | 用戶選擇（顯示所有選項） |
| 狀態 (status) | 用戶選擇（顯示所有選項） |
| 截止日期 (dueDate) | 用戶選擇（顯示所有選項） |
| 負責人 (assignees) | 合併所有負責人（全部保留） |
| 留言 (comments) | 全部保留 |
| 依賴 (dependencies) | 全部保留（需檢測循環） |

### 3.2 衝突處理

當多個任務的同一欄位有不同的值時，視為衝突：

```typescript
interface FieldConflict {
  field: "title" | "description" | "priority" | "status" | "dueDate" | "assignees";
  values: { taskId: string; value: any }[];  // 各任務的值
  suggestedValue: any;                         // 建議值（預設邏輯）
}
```

**預設建議值邏輯：**
- `priority`: 最高優先級
- `status`: 最「前面」的狀態（BACKLOG → DONE）
- `dueDate`: 最早的截止日期
- `assignees`: 合併所有負責人（不衝突）

## 4. 商業規則

### 4.1 合併條件
- 只能合併同一專案內的任務
- 最少需要 2 個任務
- 最多一次合併 10 個任務

### 4.2 合併後
- 來源任務全部刪除
- 建立新的合併任務
- 所有子任務會隨來源任務刪除（重要！需提醒用戶）
- 依賴關係需重新驗證（防止循環）

### 4.3 依賴處理
```
如果任務 A 依賴 任務 B，任務 C 依賴 任務 D：

合併 A + C → 新任務 E
合併 B + D → 新任務 F

結果：E 依賴 F（自動重建）
```

### 4.4 循環檢測
合併前檢查：
1. 列出所有來源任務的依賴
2. 檢查合併後是否會形成循環
3. 若會形成循環，返回錯誤

## 6. API 實作要點

### 6.1 預覽端點
```typescript
// POST /api/tasks/merge-preview
// 1. 取得所有任務
// 2. 偵測衝突
// 3. 產生預設建議值
// 4. 串接描述
```

### 6.2 合併端點
```typescript
// POST /api/tasks/merge
// 1. 驗證所有任務存在且同專案
// 2. 檢測依賴循環
// 3. 在 transaction 中：
//    - 建立新任務（含所有合併後的 assignees）
//    - 複製 comments 到新任務
//    - 重建依賴關係
//    - 刪除來源任務（級聯刪除子任務）
```

### 6.3 Transaction 範例
```typescript
await prisma.$transaction(async (tx) => {
  // 1. 建立合併後的任務
  const mergedTask = await tx.task.create({ ... });
  
  // 2. 複製留言
  for (const taskId of sourceTaskIds) {
    const comments = await tx.taskComment.findMany({ where: { taskId } });
    for (const comment of comments) {
      await tx.taskComment.create({
        data: { ...comment, taskId: mergedTask.id }
      });
    }
  }
  
  // 3. 重建依賴（檢測後）
  // ... 依賴重建邏輯
  
  // 4. 刪除來源任務（子任務會級聯刪除）
  await tx.task.deleteMany({ where: { id: { in: sourceTaskIds } } });
});
```

## 8. 測試案例

| 案例 | 預期行為 |
|------|----------|
| 合併 2 個無衝突任務 | 直接合併成功 |
| 合併有不同優先級的任務 | 顯示衝突，讓用戶選擇 |
| 合併任務有循環依賴 | 返回錯誤，不執行合併 |
| 合併任務有子任務 | 子任務隨之刪除，需警告用戶 |
| 合併不同專案的任務 | 返回錯誤 |
| 合併超過 10 個任務 | 返回錯誤 |
