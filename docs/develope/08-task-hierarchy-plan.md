# 任務階層與重組功能 - 實作總覽

## 1. 計畫目標

本計畫旨在擴展現有任務管理系統，支援以下高階功能：

| 功能 | 說明 |
|------|------|
| **父子任務** | 任務可擁有子任務，形成樹狀階層結構 |
| **任務合併** | 將多個任務合併為單一任務，保留所有資料 |
| **任務拆分** | 將單一任務拆分成多個獨立任務 |
| **轉為專案** | 將選定的任務轉換為新專案 |

## 2. 使用者決策

| 需求 | 選擇 |
|------|------|
| 任務合併衝突處理 | 保留所有資料，合併後顯示所有 assignees，衝突欄位讓用戶選擇 |
| 任務拆分方式 | 手動分割內容，用戶在文字編輯器中自行標記分割點 |
| 轉為專案後原任務處理 | 移入新專案，原任務從原專案刪除並移到新專案 |
| 子任務結構 | 需要支援，任務可擁有自己的子任務，形成樹狀結構 |

```

────────────────    ───────────────    ───────────────    ───────────────
1. Schema 新增       1. 父子任務 API     1. TaskTreeView     1. 商業邏輯整合
2. 遷移腳本          2. 合併 API         2. 合併對話框       2. 權限檢查完善

                    4. 轉專案 API        4. 轉專案對話框     4. 測試覆蓋
                                           5. Kanban 增強
```

## 5. 技術風險與緩解

| 風險 | 緩解方案 |
|------|----------|
| 循環依賴（合併後） | 合併前檢測，阻止並提示用戶 |
| 跨專案依賴複雜化 | 建立 ExternalTaskDependency 模型追蹤 |
| UI 複雜度提升 | 分階段交付，先做核心再逐步增強 |
| 資料遷移 | 撰寫向後相容的 migration script |
